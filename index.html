RupeeRisk
India Macro & Geopolitical Forex Intelligence Platform
Time Series ARIMA/ARIMAX Ensemble Geopolitics Streamlit
Live App → GitHub →
The Problem
Forex models treat the rupee as a statistical process and ignore what actually moves it.
Oil supply shocks, trade route disruptions, risk-off sentiment - the geopolitical signals that cause large, sudden moves in USD/INR are almost universally absent from existing forecasting tools. No one was combining macro indicators with geopolitical tension scores in a rigorous, deployment-ready way.
The Approach
15 geopolitical indicators. 9 models. A 200-week rolling out-of-sample window.
I engineered 15 tension indicators across 4 transmission channels (Oil Supply, Direct FX, Risk-Off, Trade Route) using STL decomposition and 1-week exogenous lags to eliminate look-ahead bias. After an ACF/PACF audit confirmed no meaningful seasonality, I replaced SARIMA with ARIMA(1,1,0) - the model the data actually supported - and benchmarked all 9 models against a naive random walk using Theil's U.

02
IndiaShield
Strategic Defence Intelligence & Analytics Platform
SIPRI Data Event Study Composite Index Python Plotly
GitHub →
The Problem
India's defence data existed in fragments. No one had connected it into one analytical picture.
Budget allocations, arms import flows, equity prices, and geopolitical events were scattered across SIPRI reports, NSE data feeds, and Union Budget PDFs - never synthesised. There was no quantitative way to measure how dependent India actually was on any single arms supplier, or how markets responded to border escalations.
The Approach
I built SIDS - a novel composite vulnerability metric - and ran event studies across 9 escalations.
The Strategic Import Dependency Score quantifies arms supply-chain vulnerability per supplier using import concentration, single-source risk, geopolitical stability, and domestic substitution capacity. Russia scored SIDS = 72 (Critical) post-2022. The event study tracked cumulative alpha for 6 NSE defence equities across Uri, Balakot, and Galwan over a ±30 trading-day window - HAL and BEL generated positive alpha while Nifty 50 declined.

03
ShramikSaathi
Indian Worker Rights AI Copilot
LLaMA 3.1 8B LoRA DPO RAG FAISS
GitHub →
The Problem
Millions of Indian workers don't know their rights - and no one has made it easy to ask.
PF/EPFO rules, income tax slabs, payslip audit rights, labour law protections - this information exists in scattered PDFs that workers in the informal economy can't navigate. The challenge wasn't building a chatbot. It was building one that gave legally grounded answers without hallucinating eligibility conditions.
The Approach
A 9-layer RAG + ReAct pipeline, and a novel metric to measure whether AI actually checks conditions before answering.
Our team built the pipeline on LLaMA 3.1 8B with cross-domain routing, a sufficiency gate, and FAISS retrieval over 311 documents. I contributed the Condition Coverage Score - a metric measuring whether eligibility conditions are verified before generation. CCS improved from 0.000 to 0.714 through LoRA SFT and DPO alignment, with zero citation fabrications on the held-out set.

04
LogiSense
Logistics Intelligence & Analytics Platform
PostgreSQL SQL Analytics HTML/JS Netlify
Live Demo → GitHub →
The Problem
Logistics companies drown in operational data they have no clean way to interrogate.
Branch performance, customer payment patterns, charge anomalies - all locked inside operational databases with no analytical layer. Smaller logistics companies especially lack tooling to turn raw transaction data into operational decisions.
The Approach
A BCNF-normalised schema, 20+ analytical queries, and a zero-backend dashboard anyone can use.
We designed a 7-table schema and wrote queries covering branch failure analysis, IQR-based charge outlier detection, payment mismatch detection, and percentile-based customer segmentation. I built the HTML/JS dashboard on Netlify with a live SQL query viewer and demo mode. It was adopted for pilot evaluation by 2 logistics companies.
